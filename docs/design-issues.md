## カテゴリ A：「書く」画面のエディタ体験

### A-1 🔴 カードギャラリーがない（初期画面が即エディタ）

**現状**
`CardBoardView` を開くと、左250pxのリストと右のエディタがいきなり表示される。
初心者が「自分の話の一覧」を俯瞰する場所がない。

**理想の動作**
- デフォルト: カードが大きく並ぶギャラリー表示（タイトル・本文プレビュー・文字数・ステータス）
- カードをクリック: 集中執筆モード（現在のエディタ）に遷移
- ヘッダーに「← カードに戻る」で戻れる

**参考レイアウト**
```
┌──────────────────────────────────────────────────────┐
│  ✏️ 書く                               [＋ 新しい話]  │
├──────────────────────────────────────────────────────┤
│                                                      │
│  ┌─────────────────┐  ┌─────────────────┐  ┌──────┐ │
│  │ 📄 第一話        │  │ 📄 第二話        │  │  ＋  │ │
│  │─────────────────│  │─────────────────│  │      │ │
│  │ 春の陽射しが窓   │  │ 翌朝、目を覚ます │  │新しい│ │
│  │ から差し込んで   │  │と、窓の外には雪  │  │話を  │ │
│  │ いた。…         │  │が積もっていた。… │  │追加  │ │
│  │                 │  │                 │  │      │ │
│  │ 800字  🟡 下書き │  │ 1,200字 ✅ 完成 │  │      │ │
│  └─────────────────┘  └─────────────────┘  └──────┘ │
│                                                      │
│  合計 2,000字  ｜  2話  ｜  [EPUB にする →]          │
└──────────────────────────────────────────────────────┘
```

**対象ファイル**
- `Views/CardBoardView.axaml` — ギャラリーモードの追加
- `ViewModels/CardBoardViewModel.cs` — `IsGalleryMode` プロパティの追加

### A-2 🔴 ツールバーが ScrollViewer に包まれている

**現状**
`EditorView.axaml:11`、`CardBoardView.axaml:12` ともにツールバーが
`<ScrollViewer HorizontalScrollBarVisibility="Auto">` で包まれている。
スクロールが必要なツールバーはデザインの失敗を示す。

**対応**
残件 `docs/feature-write-tab-toolbar-2026-03-10.md` で対処予定。
- ファイル操作（新規/開く/保存）と補助機能（校正・本にする）を `≡` メニューに格納
- WriteView のタブに ModeComboBox を昇格させて削除
- `Padding="4,2"` に統一してボタン幅を詰める

---

### A-3 🔴 エディタがただのテキストボックスに見える

**現状**
`#1E1E1E` 背景の TextBox に行番号が並ぶだけ。
本物のエディタが持つ視覚的フィードバックが何もない。

**TextBox で実装可能な改善（優先順）**

| # | 機能 | 概要 | 難易度 |
|---|------|------|--------|
| 1 | **検索・置換パネル** | Ctrl+F でスライドイン。現状この機能が完全に欠落しており最重要 | 中 |
| 2 | **現在行ハイライト** | カーソル行に薄い背景色オーバーレイ。「今どこを書いているか」が一目でわかる | 中 |
| 3 | **タイプライターモード** | カーソル行が常に画面中央に来るよう ScrollViewer を自動調整 | 中 |
| 4 | **原稿用紙換算** | ステータスバーに「400字詰め ○枚」を追加表示 | 小 |
| 5 | **目標文字数バー** | 「1,000字 / 2,000字 目標」プログレスバー。設定で目標を指定できる | 小 |
| 6 | **フォーカスモード** | Ctrl+Shift+F でツールバー・行番号を隠し、文章だけを全画面表示 | 小 |
| 7 | **段落ジャンプ** | 見出し行（`［＃大見出し］` など）一覧から該当行へジャンプ | 中 |
| 8 | **選択文字数表示** | テキスト選択中にステータスバーへ「5文字選択中」を表示 | 小 |

**TextBox では実装不可（AvaloniaEdit 必須）**

| 機能 | 理由 |
|------|------|
| 注記のシンタックスハイライト | TextBox はテキスト全体に単一スタイルしか適用できない |
| ルビのインライン表示 | 同上 |
| 章区切りを罫線に変換 | 同上 |

> **AvaloniaEdit 再挑戦について**
> 過去の不採用理由: Fluent テーマ未登録 + IME インライン入力非対応
> Fluent テーマ問題は `App.axaml` に StyleInclude を追加すれば解決可能。
> IME 問題は Windows 固有であり、根本解決には AvaloniaEdit 側の対応待ち。
> シンタックスハイライトの価値は高いため、将来的な再挑戦は検討の余地がある。

### A-4 🟡 エディタのダークテーマが画面の他の部分と断絶している

**現状**
エディタ部分だけ `#1E1E1E`（VS Code ダーク）で、
ツールバー・サイドバー・ステータスバーはシステムカラー（ライト系）。
同一ウィンドウ内に 2 つの世界観が共存している。

**改善案**
- エディタ背景のデフォルトをシステムテーマに準じた色にする
- ユーザーがエディタテーマ（設定画面）で変更した場合のみ `#1E1E1E` になる
- ステータスバー（現 `#2D2D30`）もテーマに追従させる

### A-5 🟡 ステータスバーの情報が貧弱

**現状（EditorView:137）**
```
ファイル名  |  文字数: 800  |  行数: 40  |  Lint: 0 warnings
```

**理想**
```
ファイル名  |  800字（原稿用紙 2枚）  |  40行  |  5文字選択中  |  L12, C4
```

## カテゴリ B：全体のナビゲーション・構造

### B-1 🔴 PreviewView・ValidateView がサイドバーから消えた

**現状**
3層サイドバーへの移行により、PreviewView と ValidateView は
サイドバーから直接アクセス不可になった。
既存 EPUB を直接プレビュー・検証したいユーザーが迷う。

**対応**
`docs/feature-web-dnd-nav-fix-2026-03-10.md` 修正 C で対処予定。
- 「読む」ページ下部に「既存の EPUB をプレビューする →」リンクを追加
- PreviewView 内の「検証」ボタンで ValidateView に遷移（現状維持）

---

### B-2 🟡 「本にする →」の導線が弱い

**現状**
`CardBoardView` のツールバー右端に小さく配置されており、目立たない。
カード執筆 → 本にするのフローは VISION の核心なのに導線が弱い。

**改善案**
- カードギャラリー（A-1）の下部フッターに単独ボタンとして配置
  ```
  合計 2,000字  ｜  2話  ｜  [📘 本にする →]
  ```
- ツールバーからは削除（またはメニュー格納）

---

### B-3 🟡 「Preview」「縦書確認」「検証」の名称が混乱を招く

**現状**
3つの異なる機能が似た名前を持つ。

| 機能 | 現在のラベル | 問題 |
|------|-------------|------|
| WebView2 で完成 EPUB を読む | プレビュー（PreviewView） | |
| LivePreview 縦書き XHTML | Preview（ToggleButton） | 上と同じ名前 |
| epubcheck 検証 | 検証（ValidateView） | |

**対応**
`docs/feature-web-dnd-nav-fix-2026-03-10.md` でツールバーの
「Preview」→「縦書確認」へのリネームが予定済み。

## カテゴリ C：画面別の細部

### C-1 🔴 PreviewView のボトムツールバーが過密

**現状**
```
← [ページ番号] → ｜ 自分でも書いてみる → ｜ フォント▼ ｜ CSS ｜ 検証 ｜ 開く ｜ ⛶
```
7 つのコントロールが 1 行に詰め込まれており、
VISION の核心リンク「自分でも書いてみる →」が埋もれている。

**改善案**
- ページナビゲーション（← [番号] →）を上部ヘッダーバーへ移動
- 「自分でも書いてみる →」を中央 or 右上に単独で目立たせる
- 「CSS / 検証 / 開く」は `⋮` メニューに格納

---

### C-2 🟡 ボタンのサイズ・余白に統一感がない

**現状**
各 View で `Padding` の値がバラバラ。

| 場所 | Padding |
|------|---------|
| MainWindow ナビゲーションボタン | `16,12` |
| ReadView モードタブ | `16,6` |
| ツールバーボタン（一部） | `10,4` |
| ツールバーボタン（EditorView） | `4,2` |

フォントサイズも `11` / `12` / `13` / `14` / `18` が混在。

**改善案**
アプリ全体で spacing scale を定義する（例: 4px 基準）。
`App.axaml` または共有スタイルファイルに基準スタイルを定義する。

---

### C-3 🟡 CardEditorView のタイトル入力欄の区切りが薄い

**現状**
タイトル入力 TextBox とエディタ本文の区切りが
`BorderThickness="0,0,0,1"` の細い線のみ。

**改善案**
背景色の差（例: タイトル欄は若干明るい `#2A2A2C`）や
余白（Margin）を増やして視覚的な分離を明確にする。

---

### C-4 🟢 ハードコードされた日本語文字列

**現状**
一部の View に `DynamicResource` を使わない日本語文字列がある。

```xml
<!-- CardBoardView.axaml:119 -->
Content="+ 新しい話を追加"

<!-- feature-web-dnd-nav-fix の指示書内 -->
Text="URLをここにドロップできます"
```

**改善案**
`Assets/Strings.ja.axaml` にキーを追加し、`DynamicResource` に統一する。
