## 0. このドキュメントの位置づけ

VISION.mdが「何を目指すか」を定義し、このドキュメントは「どう実現するか」を定義する。

**VISION.mdからの引用（設計の基軸）:**
> 「読む人」を「作る人」に変えるアプリ。
> 人はシステムになってはいけない。

## 1. アプリの3層構造

VISION.mdで定義した3層を、具体的な機能に落とし込む。

```
層1: 読む ─── 好きな作品をEPUBで読む（入口・最大の差別化）
      ↓ 「自分でも書いてみたい」
層2: 書く ─── 楽しく創作する（執筆モード）
      ↓ 「本の形にしたい」
層3: 本にする ─ 表紙・目次・テンプレートで仕上げる（ブックエディタ）
```

**旧設計との違い:**

| 観点 | 旧（feature-app-redesign.md） | 新（本ドキュメント） |
|------|-----|-----|
| 3層の定義 | 執筆→編集→制作 | 読む→書く→本にする |
| 設計思想 | 出版業界の分業モデル | 一人で全部やる人のフロー |
| 入口 | 4つのエントリーポイント | 2つ（読む / 作る） |
| Web変換の位置 | ツールに格下げ | 層1のメイン機能（最大の差別化） |
| 楽しさの設計 | なし | 各層に感情目標を設定 |

---

## 2. 層1: 読む（Read）

### 機能
- なろう/カクヨム/青空文庫のURLからEPUB生成
- ローカルテキストファイルのEPUB変換
- 生成したEPUBのプレビュー・検証

### ユーザーの感情目標
> 「好きな作品がきれいな本になった！」

### 楽しさのポイント
- URL貼り付け → 数秒で本ができる**即時フィードバック**
- 縦書きプレビューで「本物の本」のように読める
- 変換完了時の達成感（「EPUB変換が完了しました」のトースト通知）

### 層1から層2への導線
- プレビュー画面に「自分でも書いてみる」ボタン
- 変換完了後に「この作品のような小説を書いてみませんか？」の控えめな提案
- サイドバーの「書く」が常に見える位置にある

### 注意: この層はユーザー獲得チャネル
- Atticus/Vellumにはない、このアプリ固有の入口
- 「読者」は「作者」より圧倒的に多い
- この層の品質と体験を維持・向上し続けることが最重要

---

## 3. 層2: 書く（Write）

### 機能
- テキストエディタ（AvaloniaEdit）
- リアルタイム縦書きプレビュー
- ハイブリッド記法（MD + なろう/カクヨム + 青空文庫）
- ルビ/傍点/太字のツールバー＆ショートカット
- サジェスト・自動補完
- テーマ・フォントカスタマイズ
- チートシート（コンテキスト依存型）
- テンプレートからの新規作成
- カード型執筆（話単位での管理）
- 投稿サイトエクスポート（なろう/カクヨム形式）

### ユーザーの感情目標
> 「書くのが楽しい！」「自分の文章が本みたいに見える！」

### 楽しさのポイント

**即時フィードバック:**
- 書いた文字がリアルタイムで縦書きの本の姿になる
- ルビを入れた瞬間にプレビューに反映される
- 文字数カウンターがリアルタイムで増える

**達成感の可視化:**
- 今日の文字数（ステータスバーに常時表示）
- マイルストーン通知（「10000文字突破！」「初めての改ページ！」）
- 執筆ストリーク（連続執筆日数）
- 目標文字数のプログレスバー

**発見と探索:**
- 初回起動ウィザード（テーマ選択 = 自分の書く空間を作る楽しさ）
- チートシートがコンテキストに応じて「こんな書き方もできます」を提示
- 新しい記法を使うと「使えましたね！」のフィードバック

### 「システムにならない」ための設計

| やってはいけないこと | 代わりにやること |
|-------------------|---------------|
| 記法の暗記を前提にする | ツールバー + ショートカット + サジェストで記法を知らなくても使える |
| 設定項目を全部見せる | モード選択（なろう/カクヨム/汎用）で全自動。詳細は隠す |
| 正しい手順を強制する | どこからでも書き始められる。テンプレートは任意 |
| 変換エラーで止める | エラーは自動修復。どうしても直せない場合のみ通知 |

---

## 4. 層3: 本にする（Publish）

### 機能
- 書誌情報（タイトル、著者、シリーズ）
- 表紙画像の設定
- 目次の自動生成・手動編集
- 章構成の並び替え（ドラッグ＆ドロップ）
- 挿絵の配置
- フォーマッティングテンプレート
- 奥付
- EPUB出力（ワンクリック）
- 将来: PDF出力、漫画対応

### ユーザーの感情目標
> 「自分の本ができた！」「プロっぽい！」

### 楽しさのポイント

**即時フィードバック:**
- 表紙画像を設定した瞬間にプレビューに反映
- テンプレートを選ぶとリアルタイムで見た目が変わる
- 章を並べ替えると目次が自動更新される

**達成感:**
- 「EPUB変換完了！」の通知
- 出力されたEPUBのプレビュー（自分の本を「読者として」読む体験）
- ファイルサイズ・ページ数の表示（「全128ページの本ができました」）

### 「一人で全部やる」ための設計

旧設計では「執筆→編集→制作」の3画面を行き来する必要があった。
新設計では**書きながら本にする**体験を目指す。

```
旧設計（分業モデル）:
  執筆画面 → 編集画面 → 制作画面 → 変換 → プレビュー
  （5ステップ、3つの画面を行き来）

新設計（一人で全部やるモデル）:
  書く画面（プレビュー付き）→ 仕上げ → 変換
  （3ステップ、自然なフロー）
```

**具体的なUI:**

層2（書く）と層3（本にする）は**同じ画面の2つのモード**:
- 「執筆モード」: テキストエディタ + プレビュー（層2）
- 「仕上げモード」: ブックエディタ（層3）
- タブまたはトグルで切替
- 切替ても原稿データは同一

これにより「執筆→仕上げ→変換」が1つの画面内で完結する。

## 5. サイドバー構成（メニュー設計）

```
【メインエリア】
  📖 読む ──────── URL→EPUB変換、ローカル変換（層1）
  ✏️ 書く ──────── 執筆モード（層2）
  📘 本にする ──── 仕上げ（層3）

【下部固定】
  ⚙️ 設定
```

**旧設計との違い:**
| 旧 | 新 | 理由 |
|----|-----|------|
| ローカル変換（トップ） | 「読む」に統合 | 変換はメイン機能ではなく入口 |
| Web変換 | 「読む」に統合 | URL変換は最大の差別化 |
| プレビュー（独立） | 各画面に内蔵 | 書きながら/仕上げながら見る |
| 検証（独立） | 変換時に自動実行 | ユーザーが手動で検証する必要をなくす |
| 執筆 | 「書く」 | より直感的な名前 |
| 編集（NEW） | 「本にする」 | 「編集」は曖昧。ゴールを名前にする |

**3つしかないのがポイント。** 「何をすればいいか」が一目でわかる。

## 6. データモデル（BookProject）

旧設計のBookProjectモデルは技術的に堅実なので**継承する**。
ただし、楽しさの要素（統計、ゲーミフィケーション）を追加。

```csharp
public sealed class BookProject
{
    // ── メタデータ ──
    public string Title { get; set; } = "";
    public string Author { get; set; } = "";
    public string? CoverImagePath { get; set; }
    
    // ── 構成 ──
    public List<BookElement> Elements { get; set; } = [];
    
    // ── 制作設定 ──
    public string CssTemplateId { get; set; } = "default";
    public string ConversionProfileId { get; set; } = "default";
    
    // ── 統計・楽しさ（NEW） ──
    public int TotalWordCount { get; set; }
    public int TargetWordCount { get; set; }            // 目標文字数（任意）
    public DateTime CreatedAt { get; set; }
    public DateTime LastWrittenAt { get; set; }
    public List<WritingSession> Sessions { get; set; } = [];  // 執筆履歴
    public List<string> Achievements { get; set; } = [];      // 達成バッジ
}

/// <summary>執筆セッションの記録</summary>
public sealed class WritingSession
{
    public DateTime Date { get; set; }
    public int WordsWritten { get; set; }
    public TimeSpan Duration { get; set; }
}
```

BookElement、TextElement、ChapterElement、ImagePageElement、IllustrationElement は
旧設計のものをそのまま使用する（変更なし）。

## 8. 未解決の設計課題

旧設計から引き継ぎ、ビジョンの観点で再評価。

| # | 課題 | ビジョン観点での判断 |
|---|------|-------------------|
| 1 | プロジェクトファイル形式 | ディレクトリ推奨。クラウド同期フォルダとの相性が良い |

| 4 | プレビューのタイミング | カード単位で即時プレビュー。即時フィードバックが最重要 |
