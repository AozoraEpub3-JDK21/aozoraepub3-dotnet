# アプリ全体機能再設計 — 執筆・編集・制作の3層分離

> 2026-03-08 作成
> Phase E7 設計の見直し: 「カード執筆」「編集（ブックエディタ）」「制作」の責務分離

---

## 0. 設計の動機

元の Phase E7 設計では「カード型執筆モード」に編集機能を混在させていた。
ユーザーのフィードバックにより、以下の3つは **明確に分離すべき** と判明:

| 層 | 誰のため | 何をする |
|----|---------|---------|
| **執筆** | 作者（初心者〜上級者） | テキストを書く |
| **編集** | 編集者・構成担当 | 書籍としての構成を組み立てる |
| **制作** | デザイナー・仕上げ担当 | 見た目・レイアウトを整える |

これは出版業界のワークフローと一致する:
`執筆（著者）→ 編集（編集者）→ 制作（DTP）→ 校正 → 出版`

---

## 1. 改訂後のアプリ全体マップ

```
サイドバー:
  ├─ ローカル変換          ← 既存（上級者向け一括変換）
  ├─ Web変換              ← 既存
  ├─ 執筆 ─────────────── ← テキストを書くことに集中
  │   ├─ 一括モード        ← 現行（1テキスト=1冊、上級者向け）
  │   └─ ガイド付きモード   ← NEW: カード型（初心者向け）
  ├─ 編集 ─────────────── ← NEW: 書籍構成の組み立て
  │   （ブックエディタ）
  ├─ プレビュー            ← 既存
  ├─ 検証                 ← 既存
  └─ 設定                 ← 既存
```

## 2. 各層の責務定義

### 2-1. 執筆（Writing）— 作者のための空間

**コンセプト:** 「書くこと」だけに集中できる場所。見た目は気にしない。

| 機能 | 一括モード | ガイド付きモード |
|------|:---------:|:--------------:|
| テキスト入力 | ○ | ○ |
| リアルタイムプレビュー | ○ | ○ |
| ルビ/傍点/太字の記法支援 | ○ | ○ |
| チートシート | ○ | ○ |
| 文字数カウント | ○ | ○ |
| テンプレートから新規作成 | ○ | ○ |
| カード分割（話単位） | × | ○ |
| ステータス管理（下書き/完了） | × | ○ |
| 目標文字数・進捗表示 | × | ○ |
| 構成の並べ替え | × | △（章内のみ） |

**ガイド付きモードが提供するもの:**
- 「まずタイトルを入力しましょう」→「次に著者名を」→「あらすじを書きましょう」
  のようなステップバイステップの誘導
- カード単位で「何を書くべきか」が明確
- 1話ずつ書ける安心感（なろう/カクヨム投稿者に馴染み深い）

**執筆が関与しないもの:**
- ❌ 表紙画像の設定
- ❌ CSSテンプレートの編集
- ❌ 目次の構成
- ❌ 挿絵の配置
- ❌ ページレイアウト

### 2-2. 編集（Editing）— ブックエディタ

**コンセプト:** 書かれた原稿を「本」として組み立てる場所。出版社の編集者の仕事。

**入力:** 執筆で作られたテキスト（一括 or カード）、または既存のテキストファイル

**編集が担う機能:**

| カテゴリ | 機能 | 説明 |
|---------|------|------|
| **書誌情報** | タイトル・著者・シリーズ設定 | メタデータ入力フォーム |
| **表紙** | 表紙画像の選択・トリミング | 画像プレビュー付き |
| **目次** | 目次の自動生成・手動編集 | ドラッグ＆ドロップで並替え |
| **章構成** | 章・セクションの分割・統合・並替え | ツリービュー or カンバン |
| **挿絵** | 本文中への画像挿入位置の指定 | ドラッグ＆ドロップで配置 |
| **見出し管理** | 見出しレベルの一括変更 | 一覧表示 + 編集 |
| **ページ区切り** | 改ページ位置の視覚的な確認・編集 | プレビュー連動 |
| **まえがき/あとがき** | 前付け・後付けの追加・編集 | テンプレート付き |
| **奥付** | 発行日・版情報・連絡先 | フォーム入力 |

**ブックエディタのUI概念:**

```
┌──────────────────────────────────────────────────────────────┐
│ ブックエディタ                                                 │
├────────────────────┬─────────────────────────────────────────┤
│ ブック構成ツリー     │ 編集パネル（選択した要素に応じて変化）     │
│                    │                                         │
│ 📘 書誌情報        │  ┌─────────────────────────────────┐   │
│ 🖼️ 表紙           │  │  [選択中: 表紙]                  │   │
│ 📑 目次            │  │                                 │   │
│ 📝 まえがき        │  │  表紙画像: [選択...] [削除]       │   │
│                    │  │  ┌─────────────┐                │   │
│ ▼ 第一章 出会い     │  │  │             │                │   │
│   📄 1-1 朝の出来事 │  │  │  (プレビュー) │                │   │
│   📄 1-2 放課後    │  │  │             │                │   │
│   🖼️ 挿絵1        │  │  └─────────────┘                │   │
│                    │  │                                 │   │
│ ▼ 第二章 転機      │  │  位置: 表紙ページ                 │   │
│   📄 2-1 嵐の前    │  │  サイズ: 自動(表示領域に合わせる)   │   │
│                    │  │                                 │   │
│ 📝 あとがき        │  └─────────────────────────────────┘   │
│ 📋 奥付            │                                         │
│                    │                                         │
│ [+ 要素追加 ▼]     │  [▶ EPUB変換] [プレビュー]              │
└────────────────────┴─────────────────────────────────────────┘
```

### 2-3. 制作（Production）— CSSテンプレート＆レイアウト

**コンセプト:** 本の見た目を整える場所。DTPオペレーターの仕事。

**既存機能の拡張:**
現在のCSS Editor（Phase C で計画済み）をベースに、より視覚的な制作環境を提供。

| カテゴリ | 機能 | 説明 |
|---------|------|------|
| **CSSテンプレート** | 組み込みテンプレート選択 | 文芸/技術書/同人/漫画 |
| **フォント設定** | 本文・見出しのフォント指定 | プレビュー付き |
| **余白・行間** | ページ余白、行間、字間の調整 | スライダーUI |
| **ページレイアウト** | 段組み、ヘッダー/フッター | ビジュアルエディタ |
| **画像調整** | 挿絵のサイズ・配置・キャプション | ドラッグ操作 |
| **配色** | 見出し色、背景色（固定レイアウト時） | カラーピッカー |

**制作は「設定」の高度版:**
現在の設定画面にある変換オプション群を、視覚的なUIに昇格させるイメージ。

## 3. 漫画・画像本への対応

### 3-1. 個人漫画家のユースケース

```
漫画家のワークフロー:
  1. 漫画画像（PNG/JPG）をページ順に用意
  2. 表紙画像を指定
  3. 目次（章タイトル＝話タイトル）を設定
  4. EPUB(固定レイアウト) に変換
```

**必要な機能:**
- 画像ファイルのドラッグ＆ドロップによる一括読み込み
- ページ順の並べ替え（サムネイル付き）
- 見開き/単ページの切替指定
- 固定レイアウトEPUB出力
- 表紙・目次の自動生成

### 3-2. 編集画面での漫画対応

ブックエディタのツリーに **画像ページ** カードを追加:

```
📘 書誌情報
🖼️ 表紙（cover.jpg）
📑 目次

▼ 第1話「始まり」
  🖼️ p001.png  ← 画像ページ
  🖼️ p002.png
  🖼️ p003.png（見開き）
  🖼️ p004.png

▼ 第2話「展開」
  🖼️ p005.png
  🖼️ p006.png

📋 奥付
```

**編集パネル（画像ページ選択時）:**

```

│                                 │
│ ┌───────────┐ ┌───────────┐    │
│ │           │ │           │    │

│ │           │ │           │    │
```

## 4. データの流れ（3層間の連携）

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│   執筆       │     │   編集       │     │   制作       │
│  (Writing)   │────▶│  (Editing)   │────▶│ (Production) │
│             │     │             │     │             │
│ テキスト生成  │     │ 構成の組立   │     │ 見た目の仕上 │
│ カード管理   │     │ 挿絵配置    │     │ CSS編集     │
│             │     │ 目次/表紙   │     │ レイアウト   │
└─────────────┘     └──────┬──────┘     └──────┬──────┘
                           │                    │
                           ▼                    ▼
                    ┌─────────────┐     ┌─────────────┐
                    │ EPUB変換     │     │ プレビュー    │
                    │             │     │             │
                    └─────────────┘     └─────────────┘
```

**共通データモデル: BookProject**

すべての層が同一の `BookProject` を操作する:

```csharp
public sealed class BookProject
{
    // ── メタデータ ──
    public string Title { get; set; } = "";
    public string Author { get; set; } = "";
    public string? CoverImagePath { get; set; }
    
    // ── 構成 ──
    public List<BookElement> Elements { get; set; } = [];
    
    // ── 制作設定 ──
    public string CssTemplateId { get; set; } = "default";
    public Dictionary<string, string> CssOverrides { get; set; } = [];
    
    // ── 変換設定 ──
    public string ConversionProfileId { get; set; } = "default";
    public bool IsFixedLayout { get; set; } = false;  // 漫画向け
}

/// <summary>書籍を構成する要素の基底</summary>
public abstract class BookElement
{
    public string Id { get; set; } = Guid.NewGuid().ToString("N")[..8];
    public string Title { get; set; } = "";
    public int Order { get; set; }
}

/// <summary>テキスト要素（話、まえがき、あとがき等）</summary>
public sealed class TextElement : BookElement
{
    public string FilePath { get; set; } = "";     // カードモード時
    public string? InlineText { get; set; }         // 一括モード時
    public TextElementType Type { get; set; }
    public string Status { get; set; } = "draft";
    public int WordCount { get; set; }
}

/// <summary>章（コンテナ）</summary>
public sealed class ChapterElement : BookElement
{
    public List<BookElement> Children { get; set; } = [];
}

/// <summary>画像ページ（漫画向け）</summary>
public sealed class ImagePageElement : BookElement
{
    public string ImagePath { get; set; } = "";
    public bool IsSpread { get; set; } = false;     // 見開き
}

/// <summary>挿絵（テキスト中に配置）</summary>
public sealed class IllustrationElement : BookElement
{
    public string ImagePath { get; set; } = "";
    public string? Caption { get; set; }
    public string InsertAfterElementId { get; set; } = ""; // どの要素の後に挿入
}

public enum TextElementType
{
    Episode,        // 話（本文）
    Foreword,       // まえがき
    Afterword,      // あとがき
    Synopsis,       // あらすじ
    Colophon,       // 奥付
    Free            // フリー（EPUB非出力）
}
```

## 5. エントリーポイント（ユーザーはどこから始める？）

現在のアプリは「すでにテキストファイルがある人」向け。
新しい設計では **4つの入口** を提供:

| 入口 | ターゲット | 最初の画面 |
|------|-----------|-----------|
| 「小説を書く」 | 初心者の作者 | 執筆（ガイド付きモード） |
| 「原稿を編集する」 | テキストがある人 | 編集（ファイル/フォルダ読込） |
| 「テキストを変換」 | 上級者 | ローカル変換（既存） |
| 「画像から本を作る」 | 漫画家 | 編集（画像一括読込モード） |

## 6. 改訂版の実装優先度

```
Phase E7-A: テーマ・フォントカスタマイズ   ← すぐやる（既存機能の改善）
Phase E7-B: チートシート & テンプレート     ← すぐやる（小さい機能追加）
Phase E8:   ブックエディタ（編集機能）      ← 次にやる（最大の欠落機能）
Phase E9:   ガイド付き執筆モード           ← その次（E8のモデルに依存）
Phase E10:  制作（CSS/レイアウトの視覚編集）← 最後（E8の上に構築）
Phase E11:  漫画/画像本対応               ← E8と並行可能
```

### 各Phaseの依存関係

```
E7-A (テーマ)  ──── 独立、即着手可能
E7-B (チートシート) ── 独立、即着手可能

E8 (ブックエディタ)
  ├─ BookProject データモデル ← 全体の基盤
  ├─ 書誌情報フォーム
  ├─ 表紙設定
  ├─ 構成ツリー（ドラッグ＆ドロップ）
  ├─ 挿絵管理
  ├─ 目次生成
  └─ EPUB変換連携

E9 (ガイド付き執筆)
  └─ E8 の BookProject モデルを使用
     （カードの実体 = BookProject の TextElement）

E10 (制作)
  └─ E8 の BookProject モデルを使用
     + 既存の CSS Editor (Phase C) を拡張

E11 (漫画対応)
  └─ E8 の BookProject に ImagePageElement を追加
     + 固定レイアウト EPUB 出力
```
