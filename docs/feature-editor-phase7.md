# エディタ機能拡張 詳細設計 — Phase E7

> 2026-03-08 改訂（VISION v0.2 準拠）
> 前版からの主な変更:
>   - チートシートをコンテキスト依存型に再設計（静的5タブ→状況に応じた出し分け）
>   - カードモードを初心者カード（層2）と編集カード（層3）に分離
>   - 各タスクに感情目標を追加
>   - 初回起動ウィザードを廃止・画面自体で代替する設計に変更

---

## VISIONとの対応

```
層1: 読む／変換  ←── E7-A（テーマ）が快適な読み体験を支える
        ↓ 「✏️ 自分でも書いてみる →」（プレビュー隅の小さなリンク）
層2: 書く        ←── E7-B（チートシート）と E7-C-Lite（初心者カード）
        ↓ 「章分けしてみませんか？」（段階的な提案）
層3: 本にする    ←── E7-C-Full（編集カード）
```

---

| フェーズ | ユーザーが感じるべき感情 |

| 初めて開いたとき | 「なんか、できそう」 |
| 初めて書いたとき | 「書けた、本みたいになった！」 |
| 続けているとき | 「気づいたら積み上がってた」 |
| 本にするとき | 「本当に自分の本になった」 |

## 1. E7-A: テーマ＋プレビュー ＋ エディタカラーテーマ

### 感情目標
> 「このアプリで書くのが気持ちいい」
> 自分の好きな見た目で書けることが、書き続ける理由になる

### 1-1. 概要

`LivePreviewService.BuildXhtml` のCSS `background-color: #ffffff` をハードコードから外し、
アプリの「ライトモード/ダークモード」設定に応じてプレビュー背景が連動する。
加えて、ユーザーが好みのカラースキームを選べるようにする。

### 1-2. エディタテーマの定義

```csharp
public sealed class EditorTheme
{
    public required string Id { get; init; }
    public required string DisplayName { get; init; }
    public required string Category { get; init; }    // "light" | "dark"

    // エディタ領域
    public required string EditorBackground { get; init; }
    public required string EditorForeground { get; init; }
    public required string EditorCaretColor { get; init; }
    public required string EditorSelectionBg { get; init; }
    public required string EditorLineNumberFg { get; init; }
    public required string EditorLineNumberBg { get; init; }

    // プレビュー領域
    public required string PreviewBackground { get; init; }
    public required string PreviewForeground { get; init; }

    // フォント（エディタ用とプレビュー用は別々）
    public string EditorFontFamily { get; init; } = "BIZ UDGothic, Consolas, monospace";
    public double EditorFontSize { get; init; } = 14;
    public string PreviewFontFamily { get; init; } = "游明朝, Yu Mincho, serif";
    public double PreviewFontSize { get; init; } = 16;
}
```

### 1-3. 組み込みテーマ一覧

| ID | 名前 | カテゴリ | 備考 |
|----|------|---------|------|
| `light-default` | ライト | light | アプリデフォルト（ライトモード時） |
| `light-sepia` | セピア（長時間執筆用） | light | 目に優しい |
| `light-manuscript` | 原稿用紙 | light | 和風・縦書きを意識 |
| `dark-default` | ダーク | dark | アプリデフォルト（ダークモード時） |
| `dark-monokai` | Monokai | dark | 定番ダークテーマ |
| `dark-nord` | Nord | dark | 北欧デザイン |

### 1-4. フォント設定UI

- エディタフォントとプレビューフォントは**別々に**設定可能
- フォントファミリー: コンボボックスで選択（システムフォント一覧から）
- サイズ: スライダー（エディタ10-24pt、プレビュー12-32pt）
- 「書き」タブのツールバーにフォントサイズ +/- ボタンを配置（Ctrl++/Ctrl+-）

### 1-5. 設定の永続化

`GuiSettings` に追加:

```csharp
public string EditorThemeId { get; set; } = "";     // 空=アプリテーマ追従
public string EditorFontFamily { get; set; } = "";  // 空=テーマデフォルト
public double EditorFontSize { get; set; } = 0;     // 0=テーマデフォルト
public string PreviewFontFamily { get; set; } = "";
public double PreviewFontSize { get; set; } = 0;
```

### 1-6. 実装タスク

| # | タスク | 概要 |
|---|--------|------|
| E7-A1 | `EditorTheme` クラス実装 | テーマデータモデル |
| E7-A2 | 組み込みテーマ6種の定義 | 上記テーブルの実データ |
| E7-A3 | `LivePreviewService` にテーマ色を渡す | PreviewBackground/Foreground を動的に |
| E7-A4 | `EditorView` のスタイルをテーマから動的適用 | EditorBackground をバインド |
| E7-A5 | 設定画面にテーマ選択ドロップダウン追加 | リアルタイムプレビュー付き |
| E7-A6 | 設定画面にフォント選択UI追加 | エディタ/プレビュー別 |
| E7-A7 | `GuiSettings` にテーマ・フォント設定を追加 | 永続化 |
| E7-A8 | アプリテーマ切替時の自動追従ロジック | Light↔Dark |

## 2. E7-B: コンテキスト依存型チートシート ＋ 新規テンプレート

### 感情目標
> 「困ったときに、ちょうどいい情報がそこにある」
> 記法リファレンスを「調べる作業」にしない。書くことに集中できる状態を守る。

### 2-1. 設計思想：静的タブから脱却する

**旧設計（廃止）**: 5タブの静的パネルを常時表示
- 問題: Phase 1ユーザーに記法リファレンスは早すぎる
- 問題: 書いている最中に「記法タブ」を探す作業が発生する
- 問題: 感情的な文脈（背中押し）が一切ない

**新設計**: ユーザーの状態に応じて内容を切り替える

```
いつ                   何を出す           なぜ
─────────────────────────────────────────────────────────
初めて開いたとき        背中押し＋操作     「なんかできそう」にする
書いている最中          記法リファレンス   書くことを止めない
手が止まったとき        背中押し           また書き始められるように
新機能に触れたとき      操作ガイド         迷わせない
```

### 2-2. 状態の判定ルール（ルールベース）

```
状態: 「初回」
  条件: アプリ起動後、初めてエディタを開いたとき
  表示: 背中押しパネル → 操作ガイドパネル（タブ切替）

状態: 「執筆中」
  条件: 直近3分以内に文字入力がある
  表示: 記法リファレンス（使用中の記法カテゴリを優先表示）

状態: 「停滞」
  条件: エディタにフォーカスがあるが、5分以上文字入力がない
  表示: 背中押しパネル

状態: 「新機能」
  条件: 初めてカードモードを開いた / 初めてテーマを変えた etc.
  表示: 操作ガイドパネル（該当機能の説明）
```

**実装方針: ルールベース**
- 条件を手で定義・シンプルで予測可能
- 状態検知ベース（操作パターン分析）は将来課題（テレメトリーが必要な大規模アプリ向け）

### 2-3. 各パネルの内容

**背中押しパネル（Motivation）**

```
┌─────────────────────────────────────────┐
│                                         │
│   まず、1000字書いてみましょう。          │
│                                         │
│   原稿用紙2.5枚。                        │
│   このチートシートの文章くらいの量です。  │
│                                         │
│   ────────────────────────────          │
│   📊 今日: 0字    合計: 12,450字         │
│                                         │
└─────────────────────────────────────────┘
```

**操作ガイドパネル（How-to）**

```
┌─────────────────────────────────────────┐
│  使い方                                 │
│  ────────────────────────────           │
│  + 自分の話を書く  → カードを追加        │
│  カードをクリック  → 右側のエディタで編集 │
│  EPUB変換ボタン    → 本にする            │
│                                         │
│  [記法を見る →]                          │
└─────────────────────────────────────────┘
```

**記法リファレンスパネル（Reference）**

現在の5タブを維持しつつ、**最後に使った記法のタブを優先表示**する。
タブ構成: 基本記法 ／ 傍点 ／ アクセント ／ なろう記法 ／ カクヨム記法 ／ ショートカット

### 2-4. 表示トリガー

- ツールバーの「？」ボタン（トグル）
- F1キー
- 起動直後は自動表示（設定で「起動時は表示しない」チェック可）

### 2-5. 新規作成テンプレート

新規作成時に空白テキストではなく、構造化されたサンプルを挿入する。

| テンプレート | 内容 |
|------------|------|
| 空 | 従来の動作 |
| 小説テンプレート | 表題、著者名、あらすじ、本文サンプル（書きかけ） |
| 短編テンプレート | 表題、著者名、本文サンプル |
| 連載第1話テンプレート | 表題、著者名、1話、まとめ（次回予告） |

**小説テンプレートの例（ハイブリッド記法）:**

```

---

　「セリフはこのように」

　段落の最初は全角スペースから始まります。
　**強調**も使えます。

---

# 第一章　続きのタイトル

　この作品は続きます……

```

## 3. E7-C-Lite: 初心者カード（層2 — 書く）

### 感情目標
> 「こんなに書けた！」
> 書いた話がカードとして並ぶことで、積み上げを目で感じられる。
> 「難しそう」ではなく「これなら書ける」という入口を作る。

### 3-1. コンセプト：コルクボードメタファー

なろう/カクヨム読者は「1話・2話・3話」という単位を読者として身体で知っている。
この体験を「書く側」に転換するのが初心者カードの役割。

```
読む体験                書く体験
──────────────────────────────────
1話を読む              1枚のカードを書く
次の話へ               新しいカードを追加
全話一覧               コルクボードで並ぶ
```

**初心者カードは層2の機能**。複雑なプロジェクト管理は不要。
「書いたものが並ぶ」だけで十分。

### 3-2. 初期画面（B+C方式）

```
┌─────────────────────────────────────────────────┐
│                                                 │
│   ┌──────────────────────┐                     │
│   │ 📖 サンプル           │  ← サンプルラベルは   │
│   │ ──────────────────── │    小さく、でも明確に  │
│   │ 主人公が駅のホームで  │                     │
│   │ 見知らぬ手紙を拾う。  │  ← 書きかけにする     │
│   │ 差出人の名前は───     │   （完成品は×）       │
│   │                      │                     │
│   │  1,200字  ✏️ 下書き   │                     │
│   └──────────────────────┘                     │
│                                                 │
│              + 自分の話を書く                   │  ← 唯一の明確な次のアクション
│                                                 │
└─────────────────────────────────────────────────┘
```

設計意図:
- **空すぎない**（Bの空ボード+ヒント）→ 何をすればいいか分かる
- **サンプルがある**（Cのサンプルカード）→ 「こういうものか」と分かる
- **書きかけサンプル**→「自分でも続きが書けそう」という気持ちになる
- **ウィザードなし**→ 画面を見れば分かる設計で代替する

### 3-3. カードの構造（初心者カード）

```
┌────────────────────────┐
│ [タイトル（任意）]       │
│ ──────────────────── │
│ 本文の冒頭20文字くらい   │
│ が自動でここに表示され   │
│ る                      │
│                         │
│  1,200字  ✏️ 下書き     │
└────────────────────────┘
```

- タイトル: 任意（空でも「第N話」と自動入力）
- ステータス: 3種類のみ（✏️下書き / 📝 執筆中 / ✅ 完成）
- 文字数: リアルタイム表示
- 本文プレビュー: 冒頭20文字を自動表示

### 3-4. カードの操作

| 操作 | 方法 |
|------|------|
| カード追加 | `+ 自分の話を書く` ボタン |
| カード編集 | カードをクリック → 右側エディタ |
| カード並び替え | ドラッグ＆ドロップ |
| カード削除 | 右クリック → 削除（確認ダイアログ） |
| ステータス変更 | 右クリック → ステータス変更 |

### 3-5. 初心者カードの文字数表示（楽しさの設計）

ステータスバーに常時表示（じんわり型フィードバック）:

```
📚 全3話 / 合計 8,450字
```

節目の通知（ステータスバーにそっと2秒間表示）:
- 1,000字達成: 「🎉 1,000字！原稿用紙2.5枚です」
- 10,000字達成: 「🎉 10,000字！短編小説1本分です」
- 5話達成: 「🎉 5話！連載スタートできます」

### 3-6. 初心者カード → 編集カードへの移行トリガー

「まずこれ」が明確でも、「次がある」ことは見えていること。
以下のトリガーで編集カードへの誘導を**提案**する（強制しない）:

| トリガー | 提案のタイミング | 提案の文言 |
|----------|----------------|-----------|
| 話数 | 5話を超えたとき | 「章分けしてみませんか？」 |
| 並び替え | 3回以上並び替えたとき | 「構成を整理できます →」 |
| EPUB変換時 | 3話以上ある場合 | 「変換前に構成を確認しますか？」 |
| ユーザー選択 | いつでも | ツールバーに「編集モード →」ボタン |

提案は**却下できる**こと、**再び表示しない設定**があること。

### 3-7. 実装タスク（E7-C-Lite）

| # | タスク | 概要 |
|---|--------|------|
| CL-1 | 初心者カードデータモデル | タイトル・本文・ステータス・文字数 |
| CL-2 | カードリストビュー | コルクボード風グリッド表示 |
| CL-3 | カード追加/削除UI | ボタン＋右クリックメニュー |
| CL-4 | カードのドラッグ＆ドロップ | Avalonia DnD API |
| CL-5 | カードクリック → エディタ連携 | 右ペインのエディタと連動 |
| CL-6 | ステータス表示と変更UI | 3種のステータスアイコン |
| CL-7 | 文字数カウンター（全体・個別） | ステータスバーに常時表示 |
| CL-8 | 節目の通知 | ステータスバーに2秒間表示、OFF可能 |
| CL-9 | 初期サンプルカード | 書きかけサンプルの実データ |
| CL-10 | 移行トリガーの提案UI | 非侵襲的な提案バー |
| CL-11 | EPUB変換（複数カード結合） | カード→青空文庫テキスト→EPUB |

## 4. E7-C-Full: 編集カード（層3 — 本にする）

### 感情目標
> 「本の形になってきた」
> 章とエピソードが構造として見えることで、
> 「これは本当に本になる」という実感が生まれる。

### 4-1. コンセプト：バインダーメタファー

初心者カード（コルクボード）から自然に移行する「次の段階」。
Scrivenerのバインダーに近いが、**なろう/カクヨム作家が直感的に使える**設計。

初心者カードとの違い:

| | 初心者カード（層2） | 編集カード（層3） |
|--|--|--|
| メタファー | コルクボード | バインダー |
| 構造 | フラット（話が並ぶだけ） | 章 > 話の階層 |
| ステータス | 3種 | 4種（draft/writing/done/revision） |
| プロジェクトファイル | なし（.txt 単体） | あり（.aep3proj） |
| 目標文字数 | なし | 設定可能 |

### 4-2. プロジェクト構造

```
MyNovel.aep3proj/
├── project.json         ← プロジェクトメタデータ＋カード一覧
├── cards/
│   ├── cover.md         ← 表紙カード
│   ├── synopsis.md      ← あらすじカード
│   ├── ch01_ep01.md     ← 第1章 第1話
│   ├── ch01_ep02.md     ← 第1章 第2話
│   ├── ch02_ep01.md     ← 第2章 第1話
│   ├── afterword.md     ← あとがき
│   └── memo_01.md       ← フリーカード（EPUB未出力）
└── images/
    └── cover.jpg        ← 表紙画像
```

**project.json の構造:**

```json
{
  "title": "作品タイトル",
  "author": "著者名",
  "structure": [
    { "type": "cover",   "file": "cover.md" },
    { "type": "synopsis","file": "synopsis.md" },
    {
      "type": "chapter", "title": "第一章　出発",
      "children": [
        { "type": "episode", "file": "ch01_ep01.md",
          "title": "1-1 旅立ちの朝", "wordCount": 3200, "status": "done" },
        { "type": "episode", "file": "ch01_ep02.md",
          "title": "1-2 別れの言葉", "wordCount": 2800, "status": "writing" }
      ]
    }
  ]
}
```

### 4-4. ステータスの4種

| ステータス | 表示 | 色 |
|-----------|------|----|
| draft | ✏️ 下書き | グレー |
| writing | 📝 執筆中 | 青 |

### 4-5. EPUB変換フロー（編集カード）

```
project.json の structure から:
  1. 表紙カード    → BookInfo のタイトル・著者・表紙画像
  2. あらすじカード → 前付けページ
  3. 章カード      → 改ページ + 章見出し
  4. 話カード      → 本文（ハイブリッド記法 → 青空文庫記法 → XHTML）
  5. あとがきカード → 後付けページ
  6. フリーカード  → スキップ（excludeFromEpub: true）
```

### 4-6. 初心者カードからの移行

初心者カードの `.txt` ファイルを編集カードにインポートできる:
- 既存の話ファイルを1話ずつカードとして読み込む
- タイトルは自動生成（「第N話」）→ 後から変更可能
- データは失わない、後から戻せる

## 5. 優先度と実装順序

```
E7-A（テーマ・フォント）  ───→ 緊急度: 低、工数: 中
  ↓
E7-B（チートシート）      ───→ 緊急度: 中、工数: 中
  ↓
E7-C-Lite（初心者カード） ───→ 緊急度: 高、工数: 大
  ↓
E7-C-Full（編集カード）   ───→ 緊急度: 低、工数: 大
```

E7-A と E7-B は並行して進めることが可能。
E7-C-Lite の完成を以て「層2の体験」が成立する。
E7-C-Full は E7-C-Lite の土台の上に段階的に構築する。

## 6. 層1→層2の導線（このフェーズで実装）

> VISION の核心: 「読む人が書く人に変わる」

**実装: 変換直後のプレビュー画面に小さなリンクを追加**

```
変換直後のプレビュー画面
  右下に小さく（主張しない・でも見える）:
  「✏️ 自分でも書いてみる →」

  クリック → 初心者カードの初期画面（B+Cのコルクボード）へ
```

設計意図:
- 「機能が見えないと無いと思って使わない」→ 見える場所に置く
- 「多すぎるとどうしていいか分からない」→ 小さく一言だけ
- 「かっこよさは求めない」→ 洗練より、確実に届くことを優先
